Tales extends HCL2 with a small set of helper functions tuned for integration testing: looking up env vars, generating data, encoding payloads, signing requests, and pulling values out of text.

This page documents the helpers that are *not* matchers. For assertion matchers (`contains`, `optional`, `is_string`, etc.), see [Matchers](/docs/reference/matchers/).

## `env(name, default?)`

Reads an environment variable.

```hcl
config {
  base_url = env("BASE_URL", "http://localhost:1337")
  api_key  = env("API_KEY")           // no default — empty string if unset
}
```

| Argument | Type   | Notes                                                            |
| -------- | ------ | ---------------------------------------------------------------- |
| `name`   | string | Env var name.                                                    |
| `default`| any    | Optional fallback when the var is unset or empty. Omit for `""`. |

Common patterns:

```hcl
url = "${env("API_URL", "http://localhost:1337")}/users"
debug = env("DEBUG", "0") == "1"
```

## `generate(name)`

Calls a generator by name and returns its value. See [Generators](/docs/reference/generators/) for declaration and types.

```hcl
generate("user_email")             // string
generate("demo_person").first_name // string
```

Output is deterministic given `(seed, scenario, step, generator name, expression path)`. Two calls in the same step return *different* values — the expression path differs. Capture or store in `vars` to reuse.

## `jsonencode(value)`

Serialises any HCL value to a canonical JSON string. **Object keys are sorted alphabetically**; sets are sorted by encoded form; numbers preserve `json.Number` precision.

```hcl
vars {
  body = jsonencode({
    id   = "evt-1"
    type = "ping"
    data = { user = "qa@example.com" }
  })
}
```

The deterministic key ordering matters for any signing scheme — the same payload always produces the same byte sequence, so HMAC and SHA-256 signatures stay stable across runs.

## `url_encode(s)`

URL-encodes a string (component-style encoding, suitable for query parameters or path segments).

```hcl
url = "${config.base_url}/search?q=${url_encode("hello world")}"
//                                 → "hello%20world"
```

## `now_unix()`

Returns the current Unix timestamp (seconds, integer) as observed at evaluation time. **Non-deterministic by design.**

```hcl
vars {
  ts = now_unix()   // 1716822000
}
```

<Aside type="caution" title="Always capture wall-clock helpers into `vars`">
	`now_unix()` evaluates fresh every time it appears in an expression. If you reference it directly in both `request.headers.X-Timestamp` and inside an HMAC computation, the two will not match. Compute once with `vars { ts = now_unix() }` and reference `vars.ts` everywhere.
</Aside>

## `now_rfc3339()`

Same idea, RFC3339 UTC string format.

```hcl
vars {
  created_at = now_rfc3339()   // "2026-05-27T18:30:00Z"
}
```

## `hmac_sha256_hex(secret, message)`

Computes HMAC-SHA256 and returns the lowercase hex digest. The secret is **never** included in error messages.

```hcl
vars {
  sig = hmac_sha256_hex(config.webhook_secret, "${vars.ts}.${vars.body}")
}

request {
  headers = { X-Signature = "t=${vars.ts},v1=${vars.sig}" }
  body { raw = vars.body }
}
```

Pair with `jsonencode` for stable canonical inputs — see the [Signing webhooks](/docs/guides/signing-webhooks/) guide for a full recipe.

## `hmac_sha1_hex(secret, message)`

Computes HMAC-SHA1 and returns the lowercase hex digest. Exposed because RFC 6238 TOTP and a handful of legacy signing schemes still require it; prefer `hmac_sha256_hex` for new signing code. The secret is never embedded in error messages.

```hcl
vars {
  sig = hmac_sha1_hex(config.legacy_secret, vars.message)
}
```

## `totp(secret_base32, options?)`

Computes an RFC 6238 TOTP code from a Base32-encoded secret.

```hcl
vars {
  ts   = now_unix()
  code = totp(config.mfa_secret, {
    period    = 30
    digits    = 6
    algorithm = "SHA1"
    timestamp = vars.ts
  })
}
```

`totp(secret)` without options is equivalent to passing every option at its default value:

| Option      | Type   | Default              | Notes                                                                  |
| ----------- | ------ | -------------------- | ---------------------------------------------------------------------- |
| `period`    | number | `30`                 | Window size in seconds; must be `> 0`.                                 |
| `digits`    | number | `6`                  | Code length; must be `> 0` and `<= 10`.                                |
| `algorithm` | string | `"SHA1"`             | Only `SHA1` is supported in V1; other values raise an explicit error.  |
| `timestamp` | number | `now_unix()`         | Unix seconds; must be `>= 0`.                                          |

The default timestamp reads the wall clock, so two calls inside the same expression may straddle a window boundary. Capture `now_unix()` into a step-local `vars.ts` and pass it as `timestamp` whenever stability matters — exactly as in the example above.

The secret is Base32; the function normalizes user-friendly variants (lower/upper case, spaces, hyphens, missing padding). Invalid secrets fail with an opaque `invalid TOTP base32 secret` message — the raw input is never echoed.

## `regex_find(value, pattern, group?)`

Extracts the first regex match from a string. With `group = 0` (default) returns the full match; with `group = N` returns the N-th capture group.

```hcl
capture {
  // body contains "verification code is A1B2C3"
  code = regex_find(response.body, "verification code is ([A-Z0-9]{6})", 1)
}
```

| Argument  | Type   | Notes                                                              |
| --------- | ------ | ------------------------------------------------------------------ |
| `value`   | string | Subject string to search.                                          |
| `pattern` | string | Go regexp syntax (RE2). Compiled at evaluation time.               |
| `group`   | number | Optional capture group index (default 0 — the full match).         |

Returns an error if the pattern doesn't match the value. Wrap in `can(...)` (see [Matchers](/docs/reference/matchers/#can)) if you want a soft probe.

## Standard HCL functions

In addition to the helpers above, Tales exposes the standard HCL2 function set: `format`, `formatlist`, `join`, `split`, `lower`, `upper`, `trim`, `length`, `concat`, `merge`, `keys`, `values`, `range`, `tonumber`, `tostring`, `jsondecode`, `coalesce`, `try`, and more. See the [Terraform language docs](https://developer.hashicorp.com/terraform/language/functions) for a comprehensive reference — they share the same `go-cty` function library.
